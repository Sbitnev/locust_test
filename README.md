# Скрипт для нагрузочного тестирования с Locust

## Подготовка среды
Вам понадобится python 3.10+

Для начала установите зависимости, выполнив
```
pip install -r requirements.txt
```

## Настройка config.json

- base_url	Базовый URL тестируемого API (например, локальный сервер или стенд)
- endpoints	Список API‑эндпоинтов, которые будут тестироваться
- endpoints[].name	Название эндпоинта (для отчётов и логов)
- endpoints[].method	HTTP‑метод запроса (GET, POST, PUT, DELETE и т.д.)
- endpoints[].path	Путь к эндпоинту относительно base_url
- endpoints[].data	Тело запроса (используется для методов вроде POST)
- endpoints[].params	Query‑параметры (для GET или других запросов с параметрами)
- headers	Общие HTTP‑заголовки, которые будут добавляться ко всем запросам

## Запуск
Для запуска выполните
```
locust
```

Затем перейдите по адресу http://localhost:8089

Вы попадете в веб интерфейс, где можно указать
- Number of users (peak concurrency) Сколько виртуальных пользователей (виртуальных клиентов) Locust должен имитировать одновременно.
- Ramp up (users started/second) Скорость генерации пользователей, то есть как быстро Locust добавляет новых виртуальных пользователей.

И начать тестирование, нажав кнопку START. Для завершения тестирования нужно нажать STOP.

## Сохранение результатов
Для сохранения результатов после завершения теста перейдите в Download Data.
- Для загрузки html отчета нажмите Download Report
- Для загрузки CSV отчета нажмите Download requests CSV
